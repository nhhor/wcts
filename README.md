# What can they see?

This is a personal project meant to educate users on _some_ of the many datapoints any website could gather from its users. Hoping to encourage people to take a more proactive apporach to thier data security.

---

### Built using:
- [Vue](https://vuejs.org/)
- [Vite](https://vite.dev/)
- [VueUse](https://vueuse.org/guide/components.html)

### Deployed to https://nhhor.github.io/wcts/
- `npm run dev`
- `npm run deploy`
    - `npm run build` _(deploy includes build step.)_

---

### Scratchpad of ideas to come:

- IP address, user-agent
- Device type and operating system
- Browser name and version
- Screen resolution and display characteristics
- Cookies and other tracking identifiers
- Geolocation (city, region, country)
- Referring URL and clickstream data
- Search queries and keywords entered on the site
- Login credentials (username, email) and account details
- Personal identifiers (full name, address, phone number, date of birth)
- Payment information (credit‑card numbers, billing address)
- Social media handles or linked accounts
- Device fingerprinting data (installed fonts, plugins, time zone, language settings)
- Behavioral data (time spent on pages, scrolling patterns, mouse movements)
- Uploaded files or media content
- Preferences, settings, and personalization choices
- Location data from GPS or Wi‑Fi triangulation (if permission granted)
- Browser history or other data shared via APIs (e.g., WebRTC IP leak)
