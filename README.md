# What can they see?

This is a personal project meant to educate users on _some_ of the many datapoints any website could gather from its users. Hoping to encourage people to take a more proactive apporach to thier data security.

---

### Deployed to https://nhhor.github.io/wcts/

### Built using:

- [Vue](https://vuejs.org/)
- [Vite](https://vite.dev/)
- [VueUse](https://vueuse.org/guide/components.html)

### Terminal Commands:

- `npm run dev`
- `npm run deploy`
  - `npm run build` _(deploy includes build step.)_

## Idea Scratchpad:

- UserAgent data
- IP Address – Used to approximate the user’s location and track their internet activity.
  - Geolocation (city, region, country)
- Referring URL and clickstream data
- Browser Type & Version – Identifies the software and version being used (e.g., Chrome 120, Firefox 115).
- Operating System – Reveals the user’s device OS (e.g., Windows 11, macOS Ventura, Android 13).
- Device Information – Includes screen resolution, device model, and hardware specs.
- Cookies & Tracking Pixels – Small files stored on the user’s device to track behavior across sites.
- Session Data – Records pages visited, time spent, and actions taken during a browsing session.
- Login Credentials – Username, email, and password (if provided during registration or login).
- Personal Identifiable Information (PII) – Full name, address, phone number, or date of birth (if submitted via forms).
- Payment Information – Credit/debit card numbers, billing addresses, and transaction history (if used for purchases).
- Geolocation Data – Precise or approximate location (via GPS, Wi-Fi, or IP-based tracking).
- Search Queries & Browsing History – Records of keywords searched and pages visited (often via search engines or analytics tools).
- Device Fingerprinting – Combines multiple data points (e.g., fonts, plugins, timezone, language settings) to uniquely identify a device.
- Social Media Activity – Data from linked accounts (e.g., Facebook, Google, Twitter) if used for login or sharing.
- Email Addresses – Collected if the user subscribes to newsletters or creates an account.
- Behavioral Data – Clicks, scrolls, and interactions with ads or content (often tracked via heatmaps or analytics).
- Referral Source – The website or link that directed the user to the current site.
- Network Information – ISP (Internet Service Provider), connection type (e.g., Wi-Fi, mobile data), and proxy usage.
- Biometric Data – In rare cases, facial recognition or fingerprint data (if a site uses advanced authentication).
- Voice or Speech Data – If the user interacts via voice commands or audio inputs.
- Metadata from Uploads – EXIF data from images/videos (e.g., camera model, GPS coordinates, timestamp).
- Preferences, settings, and personalization choices
- Browser history or other data shared via APIs (e.g., WebRTC IP leak)
